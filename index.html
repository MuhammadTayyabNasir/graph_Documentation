General JSON Structure

All chart configurations follow a primary JSON structure with three main keys: Data, Config, and Appearance. Many properties are common across all charts, while some are specific to a chart family (e.g., Cartesian) or a single chart type.

Root Structure


{
  "Data": { ... },
  "Config": { ... },
  "Appearance": { ... }
}

Common Config Properties

These properties control the behavior and features of the chart.

PropertyLocationTypeDescription
presentation Config.viewModes.presentation String Required Defines the chart type to render. Must match one of the supported chart type strings (e.g., "linechart", "shapeMap").
fetchData Config.data.fetchData Boolean If true, the chart will attempt to fetch data from the API specified in Data.features.servercommunication. Defaults to false.
enableTooltips Config.features.enableTooltips Boolean Enables or disables tooltips on hover or tap. Defaults to true.
enableZoomAndPan Config.features.enableZoomAndPan Boolean Enables zooming and panning interactions. Mostly applicable to Cartesian and Map charts. Defaults to false.
initialVisibleItemCount Config.features.initialVisibleItemCount Number For Cartesian charts with zooming enabled, this sets the number of initial data points to display on the x-axis. Defaults to 5.

Common Data.graph Properties

These properties define the data source and its mapping.

PropertyTypeDescription
title String Required The title displayed at the top of the chart.
xAxisValue String Required The key within each data point object that maps to the X-axis value (or category label).
yAxisValue String Required The key within each data point object that maps to the Y-axis value.
data Array<Object> Required An array of data objects used to render the chart when not fetching from an API.
filterKeys Array<String> An array of keys from the data objects that should be available for user-driven filtering in the UI.

Common Appearance.graph Properties

These properties control the visual styling of the chart.

PropertyTypeDescription
color String The primary color for text elements like the title and axis labels. Can be a hex code (e.g., "#333333").
backgroundColor String The background color of the chart container. Can be a hex code. Defaults to transparent.
barColor String A general purpose Series Color used for elements like bars, columns, lines, and markers. Acts as a primary color for the data series itself. Can be a hex code.
width / height String The dimensions of the chart container. Can be a percentage (e.g., "100%") or pixels (e.g., "400px").
xAxisLabelIntersectAction String Cartesian How to handle overlapping X-axis labels. Values: "none", "hide", "trim", "rotate45", "rotate90", "multipleRows", "wrap".
showLegend Boolean Maps & Multi-Series Toggles the visibility of the chart's legend.
legendPosition String Maps Position of the legend. Values: "top", "bottom", "left", "right".

Line Chart

A standard chart that displays information as a series of data points connected by straight line segments. Ideal for showing trends over time.

Specific Appearance.graph Properties

PropertyTypeDescription
lineStyle String The style of the line. Supported values are "solid" or "dotted". Defaults to "dotted".

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Stock Price Over Time",
        "xAxisValue": "date",
        "yAxisValue": "price",
        "data": [
          {"date": "2023-01-01", "price": 150},
          {"date": "2023-01-02", "price": 152},
          {"date": "2023-01-03", "price": 155}
        ]
      }
    }
  },
  "Config": {
    "viewModes": {"presentation": "linechart"},
    "features": {"enableZoomAndPan": true, "initialVisibleItemCount": 3}
  },
  "Appearance": {
    "features": {
      "graph": [{"barColor": "#2196F3", "lineStyle": "solid"}]
    }
  }
}

Column Chart

Uses vertical bars to show comparisons among categories. One axis of the chart shows the specific categories being compared, and the other axis represents a discrete value.

Specific Appearance.graph Properties

PropertyTypeDescription
columnWidth Number The width of each column as a fraction of the available category space (e.g., 0.8 is 80%). Defaults to 0.8.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Monthly Profit",
        "xAxisValue": "month",
        "yAxisValue": "profit",
        "data": [
          {"month": "Jan", "profit": 500}, 
          {"month": "Feb", "profit": 750}, 
          {"month": "Mar", "profit": -200}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "columnchart"}},
  "Appearance": {
    "features": {
      "graph": [{"barColor": "#4CAF50", "columnWidth": 0.7}]
    }
  }
}

Bar Chart

A transposed version of the Column Chart, using horizontal bars. It's particularly useful when category labels are long.

Specific Appearance.graph Properties

PropertyTypeDescription
columnWidth Number The width of each bar as a fraction of the available category space (e.g., 0.8 is 80%). Defaults to 0.8.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Sales by Product",
        "xAxisValue": "product",
        "yAxisValue": "sales",
        "data": [
          {"product": "SKU-FR-8432-A", "sales": 50}, 
          {"product": "SKU-PT-9912-B", "sales": 80},
          {"product": "SKU-DE-5561-D", "sales": 95}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "barchart"}},
  "Appearance": {
    "features": {
      "graph": [{"barColor": "#FF9800", "xAxisLabelIntersectAction": "wrap"}]
    }
  }
}

Area Chart

Based on the line chart, but the area between the axis and the line is filled with color. It can be used to represent volume or total amounts over time.

This chart shares its configuration with the Line Chart and has no unique properties.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Website Users",
        "xAxisValue": "year",
        "yAxisValue": "users",
        "data": [{"year": 2021, "users": 1500}, {"year": 2022, "users": 2500}, {"year": 2023, "users": 3200}]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "areachart"}},
  "Appearance": {"features": {"graph": [{"barColor": "#673AB7"}]}}
}

Spline Chart

A type of line chart that connects data points with a smooth, curved line (a cardinal spline) instead of a straight one. Useful for data that changes in a continuous, non-linear fashion.

Specific Appearance.graph Properties

PropertyTypeDescription
lineStyle String The style of the spline. Supported values are "solid" or "dotted". Defaults to "dotted".

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Temperature Fluctuation",
        "xAxisValue": "time",
        "yAxisValue": "temp",
        "data": [{"time": "8am", "temp": 15}, {"time": "12pm", "temp": 22}, {"time": "4pm", "temp": 20}]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "splinechart"}},
  "Appearance": {"features": {"graph": [{"barColor": "#F44336", "lineStyle": "solid"}]}}
}

Range Column Chart

Uses vertical columns to display a range of values (a high and a low value) for each category, rather than a single value.

Specific Data.graph Properties

The yAxisValue property is ignored. Instead, each data point must contain specific keys for the high and low values.

PropertyTypeDescription
high Number Required in Data The key in each data object representing the highest point of the range for that category.
low Number Required in Data The key in each data object representing the lowest point of the range for that category.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Daily Temperature Range",
        "xAxisValue": "day",
        "yAxisValue": "ignored",
        "data": [
          {"day": "Mon", "low": 10, "high": 22}, 
          {"day": "Tue", "low": 12, "high": 24}, 
          {"day": "Wed", "low": 8, "high": 19}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "rangecolumnseries"}},
  "Appearance": {"features": {"graph": [{"barColor": "#00BCD4"}]}}
}

Bubble Chart

A variation of a Scatter Chart where data points are replaced with bubbles, and an additional dimension of the data is represented by the size of the bubbles.

Specific Data.graph Properties

In addition to xAxisValue and yAxisValue, each data point must have a key for bubble size.

PropertyTypeDescription
size Number Required in Data The key in each data object that determines the size of the bubble.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Market Analysis",
        "xAxisValue": "marketShare",
        "yAxisValue": "growthRate",
        "data": [
          {"marketShare": 15, "growthRate": 12, "size": 5000, "product": "A"}, 
          {"marketShare": 25, "growthRate": 5, "size": 8000, "product": "B"}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "bubbleseries"}},
  "Appearance": {"features": {"graph": [{"barColor": "#795548"}]}}
}

Waterfall Chart

Helps in understanding the cumulative effect of sequentially introduced positive or negative values. It shows how an initial value is affected by a series of intermediate values, leading to a final value.

Specific Data.graph Properties

Data points can have special boolean flags to denote them as summaries.

PropertyTypeDescription
isIntermediateSum Boolean Optional in Data If true, this data point is calculated as a subtotal of the previous values. Its own yAxisValue is ignored.
isTotalSum Boolean Optional in Data If true, this data point is the final sum of all previous operations. Its own yAxisValue is ignored.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Annual P&L Waterfall",
        "xAxisValue": "category",
        "yAxisValue": "amount",
        "data": [
          {"category": "Revenue", "amount": 5000}, 
          {"category": "COGS", "amount": -1500}, 
          {"category": "Gross Profit", "isIntermediateSum": true}, 
          {"category": "Ops", "amount": -800}, 
          {"category": "Net Profit", "isTotalSum": true}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "waterfallseries"}},
  "Appearance": {"features": {"graph": [{"xAxisLabelIntersectAction": "rotate45"}]}}
}

Multi-Series Column Chart

A column chart that can display and compare multiple data series side-by-side for each category. It requires a key in the data to group by.

Specific Data.graph Properties

Multi-series charts require a third key in the data (besides x and y axes) to group the data into distinct series. This key should be included in filterKeys.

PropertyTypeDescription
filterKeys Array<String> Required Must contain the key used for grouping data into different series (e.g., "metric", "product", "region").

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Revenue vs. Profit",
        "xAxisValue": "quarter",
        "yAxisValue": "amount",
        "filterKeys": ["metric"],
        "data": [
          {"quarter": "Q1", "amount": 1000, "metric": "Revenue"}, 
          {"quarter": "Q1", "amount": 400, "metric": "Profit"}, 
          {"quarter": "Q2", "amount": 1200, "metric": "Revenue"},
          {"quarter": "Q2", "amount": 550, "metric": "Profit"}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "multiseriescolumn"}},
  "Appearance": {"features": {"graph": []}}
}

Stacked Column Chart

A type of column chart where multiple series are stacked on top of each other in vertical columns. It shows the total magnitude for a category and the contribution of each sub-group.

Specific Data.graph Properties

Like multi-series charts, stacked charts require a third key in the data (besides x and y axes) to group the data into stacked segments. This key should be included in filterKeys.

PropertyTypeDescription
filterKeys Array<String> Required Must contain the key used for grouping data into different stacked series (e.g., "product", "region").

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Sales by Product per Quarter",
        "xAxisValue": "quarter",
        "yAxisValue": "sales",
        "filterKeys": ["product"],
        "data": [
          {"quarter": "Q1", "sales": 100, "product": "A"}, 
          {"quarter": "Q1", "sales": 150, "product": "B"}, 
          {"quarter": "Q2", "sales": 80, "product": "A"}, 
          {"quarter": "Q2", "sales": 120, "product": "B"}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "stackedcolumnseries"}},
  "Appearance": {"features": {"graph": [{}]}}
}

Hilo Chart

A simple financial chart that displays the high and low values over a period. It does not show open or close values.

Specific Data.graph Properties

The yAxisValue property is ignored. Instead, each data point must contain specific keys.

PropertyTypeDescription
high Number Required in Data The key in each data object representing the high value for the period.
low Number Required in Data The key in each data object representing the low value for the period.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Daily High/Low",
        "xAxisValue": "day",
        "yAxisValue": "ignored",
        "data": [{"day": 1, "high": 110, "low": 98}, {"day": 2, "high": 115, "low": 105}]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "hiloseries"}}
}

}

Box & Whisker Chart

A standardized way of displaying the distribution of data based on a five-number summary: minimum, first quartile (Q1), median, third quartile (Q3), and maximum. It can also show outliers.

Specific Data.graph Properties

The yAxisValue property must point to a key in the data whose value is an array of numbers. The chart will automatically calculate the quartiles, median, and outliers from this array.

PropertyTypeDescription
yAxisValue String Required The key that points to an array of numbers (e.g., "ms": [50, 65, 68, 72, ...]).

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "API Response Time",
        "xAxisValue": "server",
        "yAxisValue": "ms",
        "data": [
          {"server": "US-East-1", "ms": [60, 70, 75, 80, 82, 85, 90, 150]}, 
          {"server": "EU-West-1", "ms": [50, 65, 68, 72, 78, 80, 85, 95]}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "boxandwhiskerseries"}}
}

Error Bar Chart

A chart that visualizes the variability or uncertainty in a reported measurement. It's often used on top of another chart type like a Column or Bar chart.

Specific Data.graph Properties

Each data point must have a key that represents the error margin for its `yAxisValue`.

PropertyTypeDescription
error Number Required in Data The key in each data object that specifies the error value (e.g., standard deviation).

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Experimental Measurements",
        "xAxisValue": "measurement",
        "yAxisValue": "value",
        "data": [

Shape Map

A map that visualizes data by coloring geographical areas (shapes) based on a numeric value. It requires a GeoJSON file for the map shapes and data that links to those shapes.

Specific Data.graph Properties

PropertyTypeDescription
mapAssetPathStringRequired The local asset path to the GeoJSON file defining the map shapes (e.g., "assets/world_map.json").
mapShapeDataKeyStringRequired The key within the GeoJSON file's `properties` that matches the values from your `xAxisValue`. For example, if your data has `{"country": "Canada"}` and your GeoJSON has `properties: {"name": "Canada"}`, this key should be "name".

Specific Appearance.graph Properties

PropertyTypeDescription
colorMappersArray<Object>Required Defines how data values map to colors. Each object in the array has from (Number), to (Number), color (String), and text (String, for the legend).
legendIconType String Maps Sets the icon shape in the legend. Values: "circle" (default), "diamond", "rectangle", "triangle".
shapeFillColorStringThe default fill color for map shapes that don't match any data.
shapeStrokeColorStringThe color of the border around each map shape.
shapeStrokeWidthNumberThe width of the border around each map shape.
showDataLabelsBooleanIf true, displays the `xAxisValue` on each corresponding map shape.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "World Population",
        "xAxisValue": "country_name",
        "yAxisValue": "population_mil",
        "mapAssetPath": "assets/world_map.json",
        "mapShapeDataKey": "name",
        "data": [
          {"country_name": "United States of America", "population_mil": 331}, 
          {"country_name": "India", "population_mil": 1380}
        ]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "shapeMap"}},
  "Appearance": {
    "features": {
      "graph": [{
        "showLegend": true,
        "legendIconType": "diamond",
        "colorMappers": [
          {"from": 0, "to": 100, "color": "#FFFFCC", "text": "< 100M"},
          {"from": 101, "to": 500, "color": "#A1DAB4", "text": "101-500M"},
          {"from": 501, "to": 1500, "color": "#253494", "text": "> 500M"}
        ]
      }]
    }
  }
}

Polyline Map

Draws lines or paths on a map, defined by a series of latitude and longitude coordinates. Excellent for visualizing routes, boundaries, or connections.

Specific Data.graph Properties

PropertyTypeDescription
mapAssetPathStringRequired Path to the GeoJSON map file.
polylineDataString (JSON)Required A JSON string representing an array of routes. Each route object should have a `points` property which is an array of `[latitude, longitude]` pairs. Example: `"[{\"name\":\"Route 1\",\"points\":[[lat1,lon1],[lat2,lon2]]}]"`.

Specific Appearance.graph Properties

PropertyTypeDescription
lineColorStringThe color of the polyline. Falls back to `barColor`.
lineWidthNumberThe width of the polyline. Defaults to 3.0.

Example JSON


{
  "Data": {
    "features": {
      "graph": {
        "title": "Route to Windsor Castle",

        "mapShapeDataKey": "name",
        "polylineData": "[{\\"name\\":\\"Route from London\\",\\"points\\":[[51.5074,-0.1278],[51.4817,-0.6053]]}]",
        "data": [{"name": "Route 1", "distance": 22}]
      }
    }
  },
  "Config": {"viewModes": {"presentation": "polylineMap"}},
  "Appearance": {
    "features": {
      "graph": [{
        "lineColor": "#D50000",
        "lineWidth": 5
      }]
    }
  }
}
